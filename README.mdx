# 项目文档

## 项目概述

这是一个基于 monorepo(turbo) 架构的全栈项目,包含多个应用和一个共享的后端服务。项目使用 pnpm workspace 进行包管理。

## 项目结构

### 后端服务 (8000 端口) （部署到 vercel 中变成无服务器的边缘函数，边缘函数）:以上线（https://www.ljhboard.cn）

这个后端服务是要打包后通过单仓库导入其他项目，去解析类型调用接口的是 hono 的特点 ，具体看我博客

基于 Hono + Supabase（云数据库） 构建的后端服务(**最后是部署到 vercel 中变成无服务器的边缘函数**),位于 `server/api` 目录:

- CDN 上的 JavaScript 边缘运行时（Edge Runtime）的特点：
  - 位置优势
  - 代码在离用户最近的 CDN 节点上运行
  - 相比传统的集中式服务器，能大幅降低延迟
  - 用户请求不需要传回源服务器处理

* **目录结构**:

  - `api/` - Vercel 部署入口
  - `src/` - 后端服务代码
    - `hono/` - 路由和控制器
    - `libs/` - 工具库
    - `server/` - Supabase 数据库操作
    - `types/` - TypeScript 类型定义
    - `email/` - 邮件服务(未使用)

* **API 路由**:

  - `/api/design` - 设计相关 API
  - `/api/jebt` - 项目管理相关 API
  - `/api/note` - 笔记相关 API
  - `/api/form` - 表单相关 API
  - `/api/ai` - AI 相关 API
  - `/api/storage` - 存储相关 API

* **主要功能**:
  - 用户认证
  - 文件存储
  - 数据库操作
  - AI 对话(完成)
